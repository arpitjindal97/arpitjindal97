I'm a Site Reliability / Infrastructure Engineer at **Booking.com**, focused on building reliable systems, automating infrastructure, and turning operational complexity into simple, maintainable tools.

My work sits at the intersection of **Kubernetes**, **Docker**, **Go**, **Bash**, **observability**, and infrastructure automation. I like systems that are boring in production, easy to debug, and documented well enough that the next person does not need tribal knowledge to run them.

- **Developer/operator tooling** — small tools that remove repeated manual work and make systems easier to reason about.
- **Open source** — contributions and experiments around infrastructure, telemetry, Docker, Raspberry Pi networking, and self-hosted systems.
